# 2026-04-17

## BLS OEWS Drilldown Plan

1. **Ingest OEWS workbook**
   - Source: `data/national_M2024_dl.xlsx` (1,403 rows, national scope).
   - Preprocess script will load, normalize keys (strip dashes from OCC_CODE, pad area codes), attach metadata, and emit `dist/oe_national_2024.json` with essential fields and precomputed series-ID parts.

2. **Cache BLS lookup tables**
   - Mirror `oe.areatype`, `oe.area`, `oe.industry`, and `oe.datatype` under `data/oe_metadata/`.
   - Join during preprocessing to provide series areatype letter, 7-digit area codes, 6-digit industry codes, and workbook-column → datatype mappings.

3. **Precompute Series-ID stencil**
   - Store per-row metadata enabling client-side series ID assembly:
     ```json
     {
       "area_title": "U.S.",
       "area_type_letter": "N",
       "area_code": "0000000",
       "industry_code": "000000",
       "occ_code": "151211",
       "occ_title": "Software Developers",
       "metrics": {
         "TOT_EMP": {"value": 497800, "datatype": "01"},
         "H_MEAN": {"value": 53.83, "datatype": "03"}
       }
     }
     ```
   - Series format: `OE + U + {areatype} + {area_code} + {industry_code} + {occ_code} + {datatype}`.

4. **Browser proxy UX**
   - UI on `http://localhost:8080/ui` should support filtering by area, industry, SOC group, occupation; display point-in-time metrics; allow metric selection, show derived series IDs, trigger BLS API fetches, render D3 charts and data table, cache responses, and provide copyable series IDs and API URLs.

5. **Pipeline integration**
   - Preprocessing outputs JSON, frontend module (e.g., `src/visualizations/oe_drilldown/`) loads it, builds selectors, computes series IDs, fetches BLS data, renders charts (dark theme). Use Playwright for validation.

6. **Next steps**
   1. Acquire metadata files.
   2. Map workbook columns to datatype codes.
   3. Implement preprocessing script (`tools/prepare_oe_data.py`).
   4. Scaffold frontend visualization showing selectors and series IDs.
   5. Integrate BLS API fetch & charting.
   6. Validate via Playwright (e.g., SOC 15-1211 → `OEUN000000000000015121101`).

The plan emphasizes static JSON generation, client-side drilldown, and scalable architecture for future regional datasets.
