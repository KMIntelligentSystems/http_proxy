## Objective
Enable a code-review subagent inside pi by reusing the shipped `subagent` extension.

## Required Actions
1. Copy or symlink `examples/extensions/subagent/index.ts` and `agents.ts` into `~/.pi/agent/extensions/subagent/` (or the project-local `.pi/extensions/subagent/`) so pi auto-loads the tool.
2. Populate `~/.pi/agent/agents/` (or `.pi/agents/`) with a `reviewer.md` prompt based on the sample `examples/extensions/subagent/agents/reviewer.md`. Adjust the instructions to focus on the visualization/D3 review criteria you need.
3. Restart or `/reload` pi so the `subagent` tool is registered, then invoke it with tasks like `Use reviewer to audit src/charts` or chain it after implementation agents (e.g., `/implement-and-review ...`).
4. Keep `agentScope` at `"user"` unless the repository is trusted; if you need project-controlled agents, set `agentScope: "both"` and confirm when prompted.
5. Remember the tool streams subprocess output (tool calls, usage stats). Expand results (Ctrl+O) to read the reviewer’s markdown report.

## Notes
- Subagents run as separate `pi` processes, inheriting your environment; treat prompts as trusted code.
- Reviewer agent uses `read`, `grep`, `find`, `ls`, and read-only `bash` commands (`git diff`, `git show`).
- You can register helper commands or extend the `subagent` extension (e.g., tweak `renderResult`, add approval gates) if you want custom workflows.
- Binary `agentScope` choice defaults to "user"; if you include project agents, pi will prompt for confirmation.
