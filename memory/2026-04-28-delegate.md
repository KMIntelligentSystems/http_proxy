 How "Ask the delegate to have the research agent say hello" Works

 ### Phase 1: Tool Selection (Parent Pi — this process)

 1. The LLM receives the user prompt along with the system prompt, which includes the delegate tool definition registered by the extension. The tool description says: "Delegate a task to a specialized sub-agent with an         
 isolated context window. Available agents are defined in .pi/agents/.md."*
 2. The LLM decides to call delegate with:
   ```json
     { "agent": "research", "task": "Say hello to the user! Give a friendly greeting..." }
   ```

 ### Phase 2: execute() Fires (The fix that mattered)

 Previously, execute() was never called — the parent pi process hung before reaching it. Here's why it works now:

 3. execute() is invoked by the pi tool-call dispatcher with the params, an AbortSignal, an onUpdate callback, and a ctx object containing ctx.cwd.
 4. Agent discovery: discoverAgents(ctx.cwd) reads .pi/agents/*.md, parses frontmatter via parseFrontmatter(), and builds an AgentConfig[]. It finds research.md with:
     - name: research
     - model: claude-haiku-4-5
     - tools: read, bash, grep, find, ls
     - systemPrompt: (the markdown body — the BLS research specialist prompt)
 5. Lookup: agents.find(a => a.name === "research") matches → config is set.

 ### Phase 3: runAgent() — Spawning the Child (The 3 fixes)

 6. getPiCommand(cwd) — This is Fix #1. It resolves:
   ```
     C:/repos/http_proxy/node_modules/@mariozechner/pi-coding-agent/dist/cli.js
   ```
   Returns { command: "C:/.../node.exe", args: ["cli.js"] }.
   Before the fix, it used process.argv[1] which pointed to dist/cli.js — your custom TUI wrapper that spawns MCP servers, proxy, host, and enters InteractiveMode. The child would re-run the entire application, fight for ports 
 3000/8080, try to connect to 3 MCP servers, and hang waiting for interactive input. Now it runs the standard pi CLI — a simple headless agent runner.
 7. Args are assembled:
   ```
     node cli.js --mode json -p --no-session --no-extensions --no-skills --no-prompt-templates --no-themes --model claude-haiku-4-5 --tools read,bash,grep,find,ls --append-system-prompt /tmp/pi-sub-XXXX/prompt.md "Task: Say    
 hello..."
   ```
   Key flags (Fix #2 area — isolation):
     - --mode json → output is newline-delimited JSON events (not TUI)
     - --no-extensions → prevents the child from loading the subagents extension recursively
     - --no-session → no session file I/O
     - -p → pipe mode (non-interactive)
     - --append-system-prompt prompt.md → the BLS research specialist system prompt is written to a temp file and passed in
 8. spawn() launches the child process with stdio: ["ignore", "pipe", "pipe"] — stdin closed, stdout/stderr captured.                                                                                                              

 ### Phase 4: Child Pi Executes

 9. The child pi process starts as a headless, single-turn agent:
     - Loads model claude-haiku-4-5
     - Only has tools: read, bash, grep, find, ls
     - System prompt = the BLS research specialist instructions
     - User message = "Task: Say hello..."
 10. Haiku reads the system prompt, sees the task is just "say hello", and responds with a friendly greeting. It emits JSON events to stdout.

 ### Phase 5: Stdout Parsing (Fix #3)

 11. The parent reads stdout as a stream. Data arrives in chunks, buffered by newline splitting. Each complete line is .trim()'d before JSON.parse() — this is the whitespace fix that prevented parse errors from trailing \r on  
 Windows or leading spaces.
 12. For each parsed JSON event, the code looks for:
 ```js
   ev.type === "message_end" && ev.message?.role === "assistant"
 ```

    When found, it extracts the text content and calls onUpdate() to stream progress back to the parent.

 ### Phase 6: Completion & Cleanup

 13. proc.on("close") fires with exit code 0. The temp prompt file is deleted.
 14. runAgent() resolves with:
   ```js
     { output: "# Hello! 👋\nI'm your BLS Research Specialist...", turns: 1, exitCode: 0, stderr: "" }
   ```
 15. execute() returns a tool result to the parent LLM:
   ```js
     { content: [{ type: "text", text: "# Hello! 👋 ..." }], details: { agent: "research", turns: 1, ... } }
   ```
 16. The parent LLM (me) receives this tool result and summarizes it for the user.
