### Browser Proxy UX — Implementation Plan

 1. Requirements & Data Contract
 - Confirm preprocessing output schema (dist/oe_national_2024.json) once the ETL script lands (area, industry, SOC, occupation metadata, metrics, datatype codes).
 - Define the minimal JSON payload the UI needs at startup: taxonomy trees, metric catalog, column → datatype map, and sample precomputed series stencil.
 - Decide where supplemental lookup tables live (e.g., data/oe_metadata/*) and expose them via build step or serve directly.

 2. Host / Proxy Enhancements
 - Expand src/host.ts to serve a richer static bundle (HTML + JS + CSS) instead of the bare SVG canvas.
 - Add lightweight asset pipeline:
     - Option A: inline ES modules built with esbuild (no framework).
     - Option B: small Svelte/Vite app compiled into dist/ui/* and served by the host.
 - Ensure dist/oe_national_2024.json and cached API responses are accessible under /ui/data/... with correct CORS and cache headers.

 3. UI Layout & Components
 - Dark-theme, responsive layout (desktop-first 1280×720):
     - Left rail: cascading filters (Area hierarchy, Industry, SOC major/minor, Occupation).
     - Metric picker: pill buttons or dropdown for point-in-time stats (employment, wage percentiles, mean/median).
     - Series info panel: show assembled series ID, API URL, copy-to-clipboard control, and cached status indicator.
     - Main panel:
           - Timeseries chart placeholder (line/area) rendered in SVG via D3.
           - Secondary slot for bar chart or KPI cards (e.g., latest values vs national).
     - Data table: tabular snapshot of current selection with ability to toggle “Normalize / Growth %”.
     - Footer: BLS attribution, data vintage.

 4. State Management & Filtering Logic
 - Load base JSON on init, build lookup maps (area→children, SOC grouping, industry tree).
 - Implement deterministic filter state model:
     - Defaults: National area, All industries (code 000000), SOC broad group set to “All”.
     - Changing parents cascades to children (disable incompatible selections).
 - Compute derived metrics object and select current metric for display; show both value and datatype code.

 5. Series ID Assembly & Display
 - Use the precomputed stencil to assemble BLS series IDs on the client as the user changes filters/metrics.
 - Display:
     - Primary ID (e.g., OEUN000000000000015121103)
     - Optional “Comparison” IDs (e.g., regional vs national, percentile vs mean).
 - Provide “Copy ID” and “Copy API request JSON” buttons.

 6. BLS API Integration & Caching
 - Build small fetch service module:
     - POST to https://api.bls.gov/publicAPI/v2/timeseries/data/ with key from .env.
     - Handle retries, API quota errors, empty datasets.
 - Local cache strategy:
     - In-memory map keyed by series ID; optionally persist to localStorage.
     - Surface cache hit/miss status in UI.

 7. Visualization (D3)
 - Implement reusable chart utilities tuned for the dark palette:
     - Timeseries line chart with hover tooltip & latest value badge.
     - Optional sparkline for peer comparison (national vs selected area).
     - Bar chart for wage percentiles or employment by area (when metric supports).
 - For table, use standard HTML table with sticky header and zebra stripes; include export to CSV.

 8. Error Handling & UX Polish
 - Inline validation messages for unsupported combinations (e.g., occupation not available in industry).
 - Loading spinners for API calls, with graceful fallback text.
 - Keyboard accessibility: tab order, focus states, ARIA labels.

 9. Build & Deployment Integration
 - Update project scripts (package.json) to build the UI bundle during npm run build.
 - Ensure proxy still supports push_svg for agent-driven visuals; keep compatibility by adding a toggle to show “Canvas Mode” for legacy pushes if required.

 10. QA & Validation
 - Write Playwright smoke tests:
     - Load /ui, ensure filters populate from JSON.
     - Change filters, verify series ID updates and API JSON preview.
     - Mock BLS responses (using playwright_route) to test chart rendering without hitting the live API.
 - Add regression test to confirm national Software Developers example yields OEUN000000000000015121101.

 11. Documentation & Hand-off
 - Update design/ with UI wireframes + interaction notes.
 - Document data assumptions and APIs in docs/oe-drilldown-ui.md.
 - Capture future enhancements (regional rollouts, download CSV, compare multiple occupations timeline).

 This plan delivers a self-contained, browser-rendered UI on the proxy endpoint that fulfills the memory requirements: rich filtering, live BLS fetches, visualization, and copyable series metadata, while staying aligned with 
 the existing agent architecture.
