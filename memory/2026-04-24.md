 2. The Piecewise Quadratic Density Estimator: Theory and Implementation

 ### 2.1 Theoretical Foundation (O'Malley, JSM 2008)

 The paper by Meghan O'Malley at BLS describes a Piecewise Quadratic Density Estimator (PQDE) designed specifically for the OES survey's interval-censored wage data — where ~1.2 million establishments report employee counts   
 per wage bracket rather than exact wages.

 #### The Core Problem

 OES collects only binned counts: how many employees fall in each of 12 wage intervals. From these proportions alone, the PQDE must reconstruct a smooth probability density function. Traditional kernel density methods require 
 point data; iterative CDF methods (Peto, Turnbull) produce step functions with discontinuities. The PQDE offers a closed-form, non-iterative alternative.

 #### The Two Guiding Principles

 1. Mass preservation: The integral of the fitted curve over each bin must exactly equal the observed employment share for that bin. This ensures the smooth curve is faithful to the data.
 2. Smoothness: The curve should have no jumps or spikes at bin boundaries. Adjacent polynomial pieces must connect continuously with approximately matching slopes.

 #### The Construction (Paper §2.2.1, Steps 1–6)

 1. Plot the histogram (employment shares as bar heights)
 2. Handle the rightmost open-ended bin with a scaled exponential tail, parameterized from the two rightmost intervals
 3. Set connector points at each bin boundary — initially the average height of adjacent bars (zero if either neighbor is empty)
 4. Algebraically adjust connector points to minimize slope discontinuities between adjacent parabolas (§2.2.4 gives the closed-form formula)
 5. The leftmost bin uses a linear polynomial (two constraints: one connector point + area)
 6. Interior bins each get a quadratic polynomial f(x) = a + b·t + c·t² where t = x − x_left, determined by three constraints: left connector point, right connector point, and area

 #### The Connector-Point Smoothing Formula (§2.2.4)

 For a connector point at boundary x_M between a lower interval [x_L, x_M] with area A_L and upper interval [x_M, x_U] with area A_U:

 ```
   y_M = [ 2·(y_L/(x_M−x_L) + y_U/(x_M−x_U)) − 3·(A_L/(x_M−x_L)² + A_U/(x_M−x_U)²) ]
         / [ 1/(x_M−x_L) + 1/(x_M−x_U) ]
 ```

 This is derived by setting f'_lower(x_M) = f'_upper(x_M) — matching slopes at the join.                                                                                                                                          

 #### Negative-Region Handling (§2.2.2)

 When a small-share bin sits between large-share neighbors, the parabola may dip below zero. The paper replaces such segments with one or two line segments that preserve area while staying non-negative.

 #### Key Result (§3.2)

 Simulation across 5,000 samples shows the PQDE's mean and percentile estimates have virtually identical relative MSE to estimates computed directly from point data — even at sample sizes as small as 50. The estimator is also 
 shown to capture bimodal distributions better than kernel density methods.

 ### 2.2 Implementation in oe-drilldown.html

 The HTML file implements a variant of the PQDE that uses a global linear system rather than the paper's sequential connector-point algorithm. The mathematical effect is equivalent but the code structure differs.

 #### Data Pipeline

 ```
   computeHistogramData()          →  12 bins with edges, employment shares, widths
       ↓
   fitPiecewiseQuadraticDensity()  →  fit object: {edges, coeffs, masses}                                                                                                                                                         
       ↓
   sampleDensity()                 →  array of {x, y} points for D3 rendering
       ↓
   renderHistogram()               →  SVG bars + density area/line + tooltip + summary
 ```

 #### computeHistogramData() — Bin Construction

 ```javascript
   const HIST_EDGES = [0, 9.25, 12.00, 15.50, 19.75, 25.50, 32.75, 42.00, 54.00, 69.50, 89.50, 115.00, 140];
 ```

 These are the May 2024 OES wage interval boundaries (hourly). The last bin [$115, $140) caps the open-ended tail at $140 for display. For each bin, the code:                                                                    
 - Filters detailed occupations whose hourly mean falls in the interval
 - Sums their employment
 - Computes pct_employment = emp_in_bin / total_emp
 - Derives pdf = (pct / total_pct) / bin_width — the true probability density where bar area (not height) represents probability

 #### fitPiecewiseQuadraticDensity(bins) — The Linear System

 For n bins, each piece is f_i(t) = a_i + b_i·t + c_i·t² where t = x − edge_i. This gives 3n unknowns. The code builds a 3n × 3n linear system from three constraint types:

 ① Mass preservation (n equations) — Paper's guiding principle 1:

 ```javascript
   // ∫₀ʷ (a + b·t + c·t²) dt = mass_i
   row[idxA(i)] = w;           // a·w
   row[idxB(i)] = 0.5 * w * w; // b·w²/2
   row[idxC(i)] = (w³) / 3;    // c·w³/3
   addRow(row, masses[i]);
 ```

 This ensures each quadratic piece integrates to exactly the observed employment share — faithfully implementing the paper's area-preservation rule.

 ② Continuity at interior boundaries (n−1 equations) — Paper's connector-point continuity:

 ```javascript
   // f_i(w_i) = f_{i+1}(0)  →  a_i + b_i·w + c_i·w² = a_{i+1}                                                                                                                                                                    
   row[idxA(i)] = 1;  row[idxB(i)] = w;  row[idxC(i)] = w * w;
   row[idxA(i+1)] = -1;
   addRow(row, 0);
 ```

 ③ Slope continuity at interior boundaries (n−1 equations) — Paper's §2.2.4 slope matching:

 ```javascript
   // f'_i(w_i) = f'_{i+1}(0)  →  b_i + 2·c_i·w = b_{i+1}
   row[idxB(i)] = 1;  row[idxC(i)] = 2 * w;
   row[idxB(i+1)] = -1;
   addRow(row, 0);
 ```

 ④ Boundary conditions (2 equations) — Adapted from §2.2.3:

 ```javascript
   // f'_0(0) = 0  (zero slope at left edge)
   rowLow[idxB(0)] = 1;  addRow(rowLow, 0);

   // f'_{n-1}(w_{n-1}) = 0  (zero slope at right edge)
   rowHigh[idxB(last)] = 1;  rowHigh[idxC(last)] = 2 * wLast;  addRow(rowHigh, 0);
 ```

 This replaces the paper's exponential tail and linear first-interval with natural-spline-like endpoint conditions. The total: n + (n−1) + (n−1) + 2 = 3n equations for 3n unknowns — a square system.

 #### solveLinearSystem() — Gaussian Elimination

 A textbook partial-pivoting Gaussian elimination solver in ~40 lines of JavaScript. Returns the coefficient vector or null if the matrix is singular.

 #### evaluateDensity() and sampleDensity()

 ```javascript
   // Locate which bin x falls in, compute local offset t = x − edge_i
   const y = a + b * t + c * t * t;
   return Math.max(0, y);  // Paper's §2.2.2: clip negative regions
 ```

 sampleDensity evaluates 36 points per bin (432 total across 12 bins), producing a smooth curve for D3.

 #### D3 Rendering in renderHistogram()

 The density overlay is drawn in two layers:

 ```javascript
   // Shaded area (behind bars)
   const densityArea = d3.area()
     .x(d => x(d.x)).y0(ih).y1(d => y(d.y))
     .curve(d3.curveMonotoneX);
   g.append('path').datum(densityPoints)
     .attr('fill', 'rgba(167,139,250,.16)');  // violet, translucent
   areaPath.lower();  // push behind histogram bars

   // Curve line (on top)
   g.append('path').datum(densityPoints)
     .attr('stroke', 'var(--violet)')         // #a78bfa
     .attr('stroke-width', 2.4);

   // Peak marker
   g.append('circle')
     .attr('cx', x(densityPeak.x))
     .attr('cy', y(densityPeak.y))
     .attr('r', 4).attr('fill', 'var(--violet)');
 ```

 The Y-axis scale is expanded to accommodate whichever is taller — bar or curve:

 ```javascript
   if (densityPoints && densityPoints.length) {
     const densMax = d3.max(densityPoints, d => d.y);
     if (densMax > 0) yMax = Math.max(yMax, densMax * 1.1);
   }
 ```

 #### Interactive Toggle

 The density overlay is gated by STATE.showDensity and only active in PDF mode:

 ```javascript

     ? fitPiecewiseQuadraticDensity(bins) : null;
 ```

 The toggle pill wires directly to a re-render:

 ```javascript

     STATE.showDensity = !STATE.showDensity;
     renderHistogram();
   });
 ```

 Tooltips also incorporate the smooth value when hovering a bar:

 ```javascript

     const center = d.lo + d.binWidth / 2;

   }
 ```

 The implementation faithfully captures the paper's two core principles — mass preservation and smooth joins — while simplifying the endpoint treatment (no exponential tail, symmetric zero-slope boundaries). The global linear 
 system approach is actually more elegant than the paper's sequential construction: it enforces all constraints simultaneously in one matrix solve rather than iterating connector-point adjustments.
